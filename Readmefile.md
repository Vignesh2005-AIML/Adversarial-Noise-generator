This project explores how small, imperceptible perturbations—known as **adversarial noise**—can mislead deep learning image classification models. Using the **Fast Gradient Sign Method (FGSM)**, adversarial examples are generated at different epsilon (ε) values to evaluate the vulnerability of AI models. Additionally, five defense mechanisms are implemented and analyzed to understand their effectiveness and computational cost.

___
## **1. Project Overview**

Deep neural networks (DNNs) are highly accurate but vulnerable to adversarial noise. Even tiny pixel-level modifications—often invisible to humans—can cause misclassification.  
This project:
<ul>
<li>Generates adversarial images using **FGSM** (ε = 0.1 to 0.5)</li>
<li>Evaluates the impact of adversarial noise on model accuracy.</li>
<li>Analyzes five defense strategies:</li>
      <ul>
      <li>Adversarial Training</li>
      <li>Input Transformation</li>
      <li>- Defense Distillation</li>
      <li>Gradient Masking</li>
      <li>Randomization</li>
      </ul>
<li>Compares the robustness of the model under different attack strengths.</li>
<li>Highlights real-world risks and proposes future research directions.</li>
</ul>

## **5.Methodology**

### 1.Upload Image

User uploads any JPG/PNG image.

### 2.Preprocess the Image

- Convert to RGB
- Resize to 224 × 224
- Normalize to model’s input range ([-1, 1])
### 3.Predict Using MobileNetV2

The clean image is passed into **MobileNetV2**, which returns:
- A 1000-class probability vector
- The top-3 predicted classes
### 4.Create True Label (One-Hot)

The model’s predicted class is converted into a **one-hot vector**, used as the "true" label for an **untargeted attack**.

### 5.Compute Loss

Loss is calculated using:
```bash
Categorical Cross-Entropy
```
between:
- the one-hot true label
- the model’s prediction

### 6.Apply FGSM Attack

FGSM computes the gradient:
```bash
∇x​J(θ,x,y)
```
Then generates adversarial noise:
```bash
  xadv​=x+ϵ⋅sign(∇x​J)
 ```
### 7.Clip Values

Ensure the adversarial image remains valid:
- stays in [-1, 1]
- can be converted back to uint8
### 8.Predict Again (Adversarial Image)

MobileNetV2 classifies the perturbed image.

### 9.Display Results

Three images shown side-by-side:
- original
- perturbation
- adversarial
### 10. Compute Metrics

- SSIM
- PSNR

### 11.Download Option

Adversarial image exported as PNG.

## **6. Results Summary**📊

- Even very small ε values can flip the model prediction.
- **SSIM remains high** (close to 1.0) → the image appears unchanged to humans.
- **PSNR decreases** with higher ε → measurable distortion.
- Demonstrates the vulnerability of deep-learning models to adversarial inputs.
